# Day2 SSM 入门

# 一。认识框架

框架是一些列 jar 包的集合，框架可以说是一个半成品。 框架一个为了解决某一个领域而产生的。开发一个框架都会去解决某一方面的问题.。

本质： 就是别人写好的代码， 打成 jar 包的集合。 是软件开发中通用部分的代码的集合。

好处： 使用通用的代码， 利用框架可以加快开发效率。减少重复代码的编写。

# 框架特性

重用性：框架代码可以反复使用。

扩展性：框架能够很容易进行扩展功能。

维护性：框架需要具备好的维护特性.

解耦性：模块和模块之间或者功能和功能之间，关联性尽量的低。如果相互之间依赖很多，就是高耦合 了。

目前而言流行的框架非常多，比较常见的框架有 spring 框架、SpringMVC、Mybatis 框架、jpa 框架、hibernate 框架、struts 框架 、SpringBoot 等等。

# Day4 springsecurity 入门以及 JWT

# 1. 概述

# 1. 什么是 Spring Security 以及概念

Spring Security 是一个基于 Java 的框架，用于提供身份验证、授权、攻击防护等安全功能。它是基于 Spring 框架的扩展，可以与 Spring 应用程序无缝集成。 Spring Security 提供了一套全面的安全解决方案，包括用户认证、访问控制、密码加密、会话管理、跨站 点请求伪造（CSRF）防护、跨站脚本攻击（XSS）防护等功能。它可以轻松地集成到 Spring 应用程序中， 提供了一种简单且灵活的方式来保护应用程序的安全性。 Spring Security 的核心概念包括：

1. 认证（Authentication）：验证用户的身份，确保用户是合法的。

2. 授权（Authorization）：确定用户是否有权限执行特定的操作或访问特定的资源。

3. 过滤器链（Filter Chain）：定义了一系列的过滤器，用于处理请求并进行安全验证。

4. 用户详细信息服务（UserDetailsService）：用于加载用户的详细信息，包括用户名、密码、角色等。

5. 安全上下文（Security Context）：保存当前用户的安全信息，可以在应用程序的任何地方访问。

6. 记住我（Remember Me）：提供了一种记住用户身份的功能，使得用户在下次访问时无需重新输 入用户名和密码。

总之，使用 Spring Security 可以有效地保护应用程序的安全性，防止未经授权的访问和恶意攻击。 它是 Java 开发中常用的安全解决方案之一。

# 2. 什么是 JWT

JWT 是 JSON Web Token 的缩写，是一种用于在网络应用之间传递信息的安全方式。它由三部分组成：头部、载荷和签名。

头部包含了令牌的类型和使用的签名算法。

载荷包含了需要传递的信息，比如用户的身份信息、权限等。

签名用于验证令牌的真实性，防止被篡改。

JWT 的工作流程如下：

1. 用户通过提供凭证（如用户名和密码）进行身份验证。

2. 服务器和数据库的用户名密码验证凭证的有效性。

3. 如果正确，使用用户名 或者用户 id 并生成一个 JWT 令牌。

4. 服务器将 JWT 令牌发送给客户端。

5. 客户端将 JWT 令牌存储在本地（通常是在浏览器的本地存储或 Cookie 中）。

6. 客户端在后续的请求中将 JWT 令牌作为身份验证凭证发送给服务器。

7. 服务器验证 JWT 令牌的签名和有效期，并根据令牌中的声明进行授权。

8. 授权通过后见资源响应给浏览器。

# JWT 具有以下特点

1. 跨平台：JWT 是基于 JSON 格式的标准，可以在不同的平台和语言之间进行传递和解析。

2. 自包含：JWT 中包含了所有需要的信息，不需要再进行额外的查询操作。

3. 安全性：JWT 使用签名进行验证，防止被篡改和伪造。

4. 可扩展性：JWT 的载荷部分可以自定义，可以根据需要添加额外的信息。

总之，JWT 是一种简单、安全、灵活的身份验证和授权机制，广泛应用于 Web 应用和移动应用的身份验 证和 API 访问控制中。

# 1 vue 是什么？

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。

渐进式：意思就是一开始使用的时候，不需要掌握全部的功能特性，可以后续逐步增加，没有多做职责之外的事情.

Vue 只关注视图层，采用自底向上增量开发的设计。

Vue 上手很容易，如果您会 HTML+JS+CSS 的知识，就会很容易上手。如果您一点都不会 HTML+JS+CSS 的知识，不建议您一来就学习 Vue。

Vue 特点数据的双向绑定，MVVM 的思想，后续会介绍 MVVM。 Vue 有组件化的思想，可以把页面中可以复用的内容，抽出一个可以复用的组件。其他页面都可以进行使用。

Vue 是一个独立的社区驱动的项目，它是由尤雨溪在 2014 年作为其个人项目创建，是一个成熟的、经历了无数实战考验的框架，它是目前生产环境中使用最广泛的 JavaScript 框架之一，可以轻松处理大多数 web 应用的场景，并且几乎不需要手动优化，并且 Vue 完全有能力处理大规模的应用。

# 8.Vue 的生命周期

Vue 的生命周期从 Vue 实例初始化创建、 模板的挂载、 以及数据的修改和销毁整个过程共计四个大阶 段， 每个阶段都有两个钩子函数。 在执行到对应阶段时， 钩子函数会自动被调用。 用户可以在对应的 钩子函数中完成相应的操作。 从初始化到销毁的过程就是 Vue 的生命周期。

对应钩子函数介绍：

beforeCreate: 在 Vue 实例初始化之后，数据和事件配置之前被调用.

created: 在实例创建完成后被立即同步调用。在这一步中，实例已完成对选项的处理，意味着以下内容 已被配置完毕：数据侦听、计算属性、方法、事件 / 侦听器的回调函数。然而，挂载阶段还没开始。

beforeMount: 在挂载前被调用，render 函数被调用，生成虚拟的 dom, 真实 dom 未创建.

mounted: 挂载完成。真实 dom 树已经完成，渲染到页面。可以进行 dom 操作。

beforeUpdate：在数据发生改变后，DOM 被更新之前被调用。这里适合在现有 DOM 将要被更新之前访问 它，比如移除手动添加的事件监听器。

updated: 在数据更改导致的虚拟 DOM 重新渲染和更新完毕之后被调用。

beforeDestroy：实例销毁之前调用。在这一步，实例仍然完全可用。

destroyed: 实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。

# Day9 Vuex

# 一、 Vuex 是什么？

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 也集成到 Vue 的官方调试工具 devtools extension (opens new window) ，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。

# 二、什么是 “状态管理模式”？

状态管理模式、集中式存储管理这些名词听起来就非常高大上，让人捉摸不透。

其实，你可以简单的将其看成把需要多个组件共享的变量全部存储在一个对象里面。然后，将这个对象放在顶层的 Vue 实例中，让其他组件可以使用。

那么，多个组件是不是就可以共享这个对象中的所有变量属性了呢？

等等，如果是这样的话，为什么官方还要专门出一个插件 Vuex 呢？难道我们不能自己封装一个对象来管理吗？

当然可以，只是我们要先想想 VueJS 带给我们最大的便利是什么呢？

没错，就是响应式。
如果你自己封装实现一个对象能不能保证它里面所有的属性做到响应式呢？

当然也可以，只是自己封装可能稍微麻烦一些。不用怀疑，Vuex 就是为了提供这样一个在多个组件间共享状态的插件，用它就可以了。

# get 和 Post 的区别

1.get 类型的请求 只能发送文本 ， post 可以是任何类型的数据。

2.get 类型的请求提交数据的大小一般不超过 4K, post 理论上大小没有限制

3.get 请求参数地址栏可见，post 提交的数据地址不可见.

4.get 和 post 相比， post 请求相对更安全。

5.get 请求的效率更高。

6. 获取数据使用 get， 对于数组的保存， 修改使用 post.
