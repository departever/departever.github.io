# 构建工具 Maven 安装

# 1. 概念

Maven 项目对象模型 (POM)，可以通过一小段描述信息来管理项目的构建，报告和文档的项目管理工具软件。

# 2. 什么是构建
  • 将 java 项目中的源码， 通过清理、编译、 测试、打包、安装、 部署整个过程称之为项目的构建。
  • 清理： 清理之前编译的内容， 如字节码文件。
  • 编译： 将 java 的源文件编译成字节码文件。
  • 测试： 对代码中的核心功能通过测试工具，如 JUnit 进行测试。
  • 打包： 将生成的字节吗文件打包为 jar 或者 war 包。

# day2 SSM 入门

# 一。认识框架

框架是一些列 jar 包的集合，框架可以说是一个半成品。 框架一个为了解决某一个领域而产生的。开发一个框架都会去解决某一方面的问题.。

本质： 就是别人写好的代码， 打成 jar 包的集合。 是软件开发中通用部分的代码的集合。

好处： 使用通用的代码， 利用框架可以加快开发效率。减少重复代码的编写。

# 框架特性

重用性：框架代码可以反复使用。

扩展性：框架能够很容易进行扩展功能。

维护性：框架需要具备好的维护特性.

解耦性：模块和模块之间或者功能和功能之间，关联性尽量的低。如果相互之间依赖很多，就是高耦合 了。

目前而言流行的框架非常多，比较常见的框架有 spring 框架、SpringMVC、Mybatis 框架、jpa 框架、hibernate 框架、struts 框架 、SpringBoot 等等。

# 二。框架的认识和使用

# 1.SSM 框架

SSM 是三个框架的组合，传统的 SSM 是指的是 springmvc+spring+mybatis

传统的 SSM 需要做大量的配置，比如 springmvc 的配置，spring 的配置，mybatis 的配置，使用起来非常的麻烦的。不好用。

现在企业里面使用最多的 Springboot。

# 2. SpringBoot 认识

SpringBoot 框架是 Spring 家族的，它设计的目得是用来简化 Spring 应用的初始化搭建和开发过程的，通过简单的配置，就可以完成传统方式的功能，让开发变得更加简单，快速，方便。

面试： 为什么要用 SpringBoot 框架？

(1) SpringBoot 目前企业可以说都在使用，而且面试是必问的

(2) SpringBoot 应用可以单独的运行，可以形成很多独立的服务，符合微服务的场景

(3) SpringBoot 里面内置了 tomcat 服务器，不用单独的部署一个 tomcat

(4) SpringBoot 通过提供的很多 start，可以很方便的整合其他的框架，比如 mybatis ，redis 等

(5) SpringBoot 简化了很多配置，不像传统程序需要大量的配置

# 四。整合日志

# 1. 日志介绍

在开发中，日志是非常重要，通过日志可以记录程序运行情况，可以定位信息，可以进行排错，所以说日志是非常重要得。

在 java 中已经存在很多的日志框架，比如 JCL,SLF4j,log4j,log4j2,logback 等等.

SpringBoot 是采用 SLF4J + LogBack 的方式来实现日志.

# 2. 日志级别

使用日志的时候，是存在日志的级别，日志级别是对日志记录信息的轻重缓急的划分.

从轻到重的级别: TRACE< DEBUG < INFO < WARN < ERROR

如果设置为 INFO ，则低于 INFO 的信息都不会输出。

# 3. 日志的配置

配置日志级别是配置到 application.yml 或者 application.properties 文件中.
