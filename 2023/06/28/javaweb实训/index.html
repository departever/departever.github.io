# Day2 SSM 入门

# 一。认识框架

框架是一些列 jar 包的集合，框架可以说是一个半成品。 框架一个为了解决某一个领域而产生的。开发一个框架都会去解决某一方面的问题.。

本质： 就是别人写好的代码， 打成 jar 包的集合。 是软件开发中通用部分的代码的集合。

好处： 使用通用的代码， 利用框架可以加快开发效率。减少重复代码的编写。

# 框架特性

重用性：框架代码可以反复使用。

扩展性：框架能够很容易进行扩展功能。

维护性：框架需要具备好的维护特性.

解耦性：模块和模块之间或者功能和功能之间，关联性尽量的低。如果相互之间依赖很多，就是高耦合 了。

目前而言流行的框架非常多，比较常见的框架有 spring 框架、SpringMVC、Mybatis 框架、jpa 框架、hibernate 框架、struts 框架 、SpringBoot 等等。

# Day4 springsecurity 入门以及 JWT

# 1. 概述

# 1. 什么是 Spring Security 以及概念

Spring Security 是一个基于 Java 的框架，用于提供身份验证、授权、攻击防护等安全功能。它是基于 Spring 框架的扩展，可以与 Spring 应用程序无缝集成。 Spring Security 提供了一套全面的安全解决方案，包括用户认证、访问控制、密码加密、会话管理、跨站 点请求伪造（CSRF）防护、跨站脚本攻击（XSS）防护等功能。它可以轻松地集成到 Spring 应用程序中， 提供了一种简单且灵活的方式来保护应用程序的安全性。 Spring Security 的核心概念包括：

1. 认证（Authentication）：验证用户的身份，确保用户是合法的。

2. 授权（Authorization）：确定用户是否有权限执行特定的操作或访问特定的资源。

3. 过滤器链（Filter Chain）：定义了一系列的过滤器，用于处理请求并进行安全验证。

4. 用户详细信息服务（UserDetailsService）：用于加载用户的详细信息，包括用户名、密码、角色等。

5. 安全上下文（Security Context）：保存当前用户的安全信息，可以在应用程序的任何地方访问。

6. 记住我（Remember Me）：提供了一种记住用户身份的功能，使得用户在下次访问时无需重新输 入用户名和密码。

总之，使用 Spring Security 可以有效地保护应用程序的安全性，防止未经授权的访问和恶意攻击。 它是 Java 开发中常用的安全解决方案之一。

# 2. 什么是 JWT

JWT 是 JSON Web Token 的缩写，是一种用于在网络应用之间传递信息的安全方式。它由三部分组成：头部、载荷和签名。

头部包含了令牌的类型和使用的签名算法。

载荷包含了需要传递的信息，比如用户的身份信息、权限等。

签名用于验证令牌的真实性，防止被篡改。

JWT 的工作流程如下：

1. 用户通过提供凭证（如用户名和密码）进行身份验证。

2. 服务器和数据库的用户名密码验证凭证的有效性。

3. 如果正确，使用用户名 或者用户 id 并生成一个 JWT 令牌。

4. 服务器将 JWT 令牌发送给客户端。

5. 客户端将 JWT 令牌存储在本地（通常是在浏览器的本地存储或 Cookie 中）。

6. 客户端在后续的请求中将 JWT 令牌作为身份验证凭证发送给服务器。

7. 服务器验证 JWT 令牌的签名和有效期，并根据令牌中的声明进行授权。

8. 授权通过后见资源响应给浏览器。

# JWT 具有以下特点

1. 跨平台：JWT 是基于 JSON 格式的标准，可以在不同的平台和语言之间进行传递和解析。

2. 自包含：JWT 中包含了所有需要的信息，不需要再进行额外的查询操作。

3. 安全性：JWT 使用签名进行验证，防止被篡改和伪造。

4. 可扩展性：JWT 的载荷部分可以自定义，可以根据需要添加额外的信息。

总之，JWT 是一种简单、安全、灵活的身份验证和授权机制，广泛应用于 Web 应用和移动应用的身份验 证和 API 访问控制中。

package com.example.controller;

import com.example.common.Result;
import com.example.common.StatusCode;
import com.example.domain.MyUser;
import com.example.service.UserService;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.validation.BindingResult;
import org.springframework.validation.ObjectError;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.RestController;

import javax.annotation.Resource;
import javax.validation.Valid;

import java.util.List;

@RestController
@RequestMapping("/user")
public class UserController {
    // 根据名称来获取存放到spring容器中的对象
    // @Autowired 通过类型在spring 容器中查找对象
    @Resource(name = "userService")
    UserService userService;

    @RequestMapping(value = "/user",method = RequestMethod.GET)
    // 设置可以访问该方法的用户的权限标识， 只有该用具有该标识权限， 才可以访问
    @PreAuthorize("hasAuthority('admin')")
    public String findAllUsers(){
        System.out.println("all users ");
        return "all Users ";
    }

    @RequestMapping("/add")
    public Result<String> add(@Valid @RequestBody MyUser user, BindingResult result){
        // 判断表单验证是否正确
        if (result.hasErrors()) {
            List<ObjectError> allErrors = result.getAllErrors();
            StringBuilder sb = new StringBuilder();
            allErrors.forEach(error -> sb.append(error.getDefaultMessage() + " : " ));
            String mes = sb.toString();
            return new Result<String>(false, StatusCode.ERROR,mes);
        }

        boolean bo = userService.add(user);
        return bo ? new Result<String>(true, StatusCode.OK,"注册成功~~")
                : new Result<String>(false, StatusCode.ERROR,"注册失败~~~");
    }
}

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。

# 8.Vue 的生命周期

Vue 的生命周期从 Vue 实例初始化创建、 模板的挂载、 以及数据的修改和销毁整个过程共计四个大阶 段， 每个阶段都有两个钩子函数。 在执行到对应阶段时， 钩子函数会自动被调用。 用户可以在对应的 钩子函数中完成相应的操作。 从初始化到销毁的过程就是 Vue 的生命周期。

对应钩子函数介绍：

beforeCreate: 在 Vue 实例初始化之后，数据和事件配置之前被调用.

created: 在实例创建完成后被立即同步调用。在这一步中，实例已完成对选项的处理，意味着以下内容 已被配置完毕：数据侦听、计算属性、方法、事件 / 侦听器的回调函数。然而，挂载阶段还没开始。

beforeMount: 在挂载前被调用，render 函数被调用，生成虚拟的 dom, 真实 dom 未创建.

mounted: 挂载完成。真实 dom 树已经完成，渲染到页面。可以进行 dom 操作。

beforeUpdate：在数据发生改变后，DOM 被更新之前被调用。这里适合在现有 DOM 将要被更新之前访问 它，比如移除手动添加的事件监听器。

updated: 在数据更改导致的虚拟 DOM 重新渲染和更新完毕之后被调用。

beforeDestroy：实例销毁之前调用。在这一步，实例仍然完全可用。

destroyed: 实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。

# 5.6 Checkbox 多选框

  • 在一组备选选项中可以选择多个， 也可以一个不选。

  • 实例：

    <template>
    <div>
        <el-checkbox :indeterminate="isIndeterminate" v-model="checkAll" @change="handleCheckAllChange">全选</el-checkbox>
        <div style="margin: 15px 0;"></div>
        <el-checkbox-group v-model="checkedCities" @change="handleCheckedCitiesChange">

        <el-checkbox v-for="city in cities" :label="city" :key="city">{{city}}</el-checkbox>
        
        </el-checkbox-group>
    </div>
</template>
  <script>
    const cityOptions = ['上海', '北京', '广州', '深圳'];
    export default {
      data() {
        return {
          checkAll: false,
          checkedCities: ['上海', '北京'],
          cities: cityOptions,
          isIndeterminate: true
        };
      },
      methods: {
        // 点击全选和全不选时触发的函数
        handleCheckAllChange(val) {
          this.checkedCities = val ? cityOptions : [];
          this.isIndeterminate = false;
        },
        // 点击选项时触发的函数
        handleCheckedCitiesChange(value) {
          let checkedCount = value.length;
          this.checkAll = checkedCount === this.cities.length;
          this.isIndeterminate = checkedCount > 0 && checkedCount < this.cities.length;
        }
      }
    };
  </script>

# ECMAScript 6

# 1. 简介

​ ECMAScript 6.0（以下简称 ES6）是 JavaScript 语言的下一代标准，已经在 2015 年 6 月正式发布了。它的目标，是使得 JavaScript 语言可以用来编写复杂的大型应用程序，成为企业级开发语言。

优点:

  • 提升了 Js 编写复杂大型应用程序的能力；(这次提升，主要加入了模块化的概念，优化了一些基础语法，封装了一些方法，使得 JS 更可用于大型模块化项目的开发)

缺点:

  • 和一些浏览器还存在兼容性问题，解决办法：Bable 插件 (将 ES6->ES5) 转换器

# 2. 基本语法

ES 标准中不包含 DOM 和 BOM 的定义，只涵盖基本数据类型、关键字、语句、运算符、内建对象、内建函数等通用语法。

# 1）let 声明变量

  1. 块级作用域:

    let 实际上为 JavaScript 新增了块级作用域。

  2. 不允许重复声明

    let 不允许在相同作用域内，重复声明同一个变量。

  3. 不存在变量提升

    var 命令会发生” 变量提升 “现象，即变量可以在声明之前使用，值为 undefined 。这种现象多多少少是有些奇怪的，按照一般的逻辑，变量应该在声明语句之后才可以使用。

    为了纠正这种现象， let 命令改变了语法行为，它所声明的变量一定要在声明后使用，否则报错。

# 2）const 声明常量

  1. const 声明一个只读的常量。一旦声明，常量的值就不能改变。

  2. 一但声明必须初始化，否则会报错
